### RELATIONSHIPS:

1. users(1) → bookings(M) [user_id]
2. services(1) → bookings(M) [service_id]
3. bookings(1) → payments(M) [booking_id]
4. users(1) → payments(M) [user_id]
5. users(1) → payments(M) [verified_by]
6. bookings(1) → reviews(1) [booking_id]
7. users(1) → reviews(M) [user_id]
8. services(1) → reviews(M) [service_id]
9. users(1) → notifications(M) [user_id]
10. users(1) → activity_logs(M) [user_id]
11. users(1) → activity_logs(M) [admin_id]
12. payments(1) → payment_logs(M) [payment_id]
13. users(1) → payment_logs(M) [performed_by]

### CARDINALITY NOTES:
- 1:M = One to Many
- 1:1 = One to One
- M:M = Many to Many (handled through junction tables)

### DRAW.IO INSTRUCTIONS:
1. Create rectangles for each entity
2. List attributes inside rectangles
3. Mark primary keys with (PK)
4. Mark foreign keys with (FK)
5. Draw lines between related entities
6. Add cardinality symbols (1, M) at line ends
7. Use different colors for entity types:
   - Blue: Core entities (users, services, bookings)
   - Green: Transaction entities (payments)
   - Yellow: Support entities (reviews, notifications)
   - Gray: Audit entities (logs)
